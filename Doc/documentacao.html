1. Proposta do sistema

O Clube Social IBTP é um sistema web desenvolvido para a disciplina de Criação de Sites Dinâmicos, com o objetivo de gerenciar o cadastro de membros e os eventos promovidos pelo clube.

O sistema possui dois níveis de acesso:

  • Membro (nível 1): pode visualizar seus dados e a lista de eventos do clube.
  • Administrador (nível 2): além de tudo que o membro pode fazer, também pode cadastrar, atualizar e excluir eventos, e excluir usuários cadastrados.

Tecnologias utilizadas: HTML, CSS, JavaScript, PHP (PDO) e MySQL/MariaDB.

O acesso ao sistema é feito por meio de login (ID + senha), com controle de sessão via $_SESSION e verificação de nível de acesso em todas as páginas internas (verifica.php).

2. Regras de negócio

  • Cada usuário possui exatamente um cargo (Membro ou Administrador), definido pela coluna cargo_id.
  • O cargo determina o que o usuário pode fazer no sistema:
    • Administradores podem cadastrar, atualizar e excluir eventos, e excluir usuários.
    • Membros podem apenas visualizar a lista de eventos e seus próprios dados.
  • Para acessar o sistema, o usuário precisa estar cadastrado e informar ID e senha corretos.
  • A senha é armazenada no banco de dados criptografada com MD5.
  • Todo novo cadastro recebe automaticamente o cargo de Membro (cargo_id = 1), salvo indicação contrária.
  • Um cargo não pode ser excluído se houver usuários vinculados a ele (restrição ON DELETE RESTRICT).

Relação entre as classes: as três classes são independentes e se relacionam apenas por meio das tabelas do banco de dados (Usuario ↔ Cargos via cargo_id; Eventos é gerenciada por usuários do tipo Administrador através da classe Cargos/Eventos).

4. Diagrama de Entidade-Relacionamento (Tabelas)

Nome do banco de dados: clube

4.1 Tabela: cargos

CampoTipoDescrição
idINT (PK)Identificador do cargo
nomeVARCHAR(255)Nome do cargo (Membro, Administrador)

4.2 Tabela: usuarios

CampoTipoDescrição
idVARCHAR(255) (PK)Identificador do usuário, definido por ele mesmo
nomeVARCHAR(255)Nome completo do membro
senhaVARCHAR(32)Senha criptografada (MD5)
sexoCHAR(1)M ou F
cargo_idINT (FK → cargos.id)Cargo do usuário

4.3 Tabela: eventos

CampoTipoDescrição
id_eventoINT (PK, AUTO_INCREMENT)Identificador do evento
nomeVARCHAR(255)Nome do evento
_dataDATETIMEData e hora do evento
_localVARCHAR(255)Local do evento

4.4 Relacionamentos

  • cargos (1) ──── (N) usuarios: um cargo pode estar associado a vários usuários, mas cada usuário possui apenas um cargo (FK usuarios.cargo_id → cargos.id).
  • eventos: tabela independente, sem relacionamento direto de chave estrangeira com as demais — o controle de quem pode gerenciá-la é feito pela aplicação (nível de acesso do usuário logado).
