1. Apresentacao do sistema

O sistema Clube Social IBTP e uma aplicacao web desenvolvida como trabalho da disciplina de Criacao de Sites Dinamicos, com foco na integracao de banco de dados com PHP.

O sistema permite o cadastro e login de membros do clube, e o gerenciamento de eventos. Dependendo do cargo do usuario, ele pode criar ou excluir eventos, ou apenas visualiza-los.

Tecnologias utilizadas: HTML, PHP e MySQL.

2. Regras de negocio

  • Cada membro possui apenas um cargo por vez.
  • O cargo determina o que o usuario pode fazer no sistema:
    • Membros com cargo de Admin podem criar e excluir eventos e excluir usuarios.
    • Membros com cargo comum so podem visualizar os eventos.
  • Para acessar o sistema o usuario precisa estar cadastrado e informar ID e senha corretos.
  • A senha é armazenada no banco de dados.

3. Classes

As classes estao na pasta model/.

3.1 Classe Usuario

Arquivo: model/usuario.php

Representa um membro cadastrado no clube.

Atributos:

  • $id - identificador unico do usuario (texto, definido pelo proprio usuario)
  • $nome - nome completo do membro
  • $sexo - sexo do membro (M ou F)
  • $senha - senha de acesso

Metodos:

  • __construct($wnome, $wsexo, $wid, $wsenha) - construtor da classe, inicializa os atributos
  • cadastrar() - insere um novo usuario no banco de dados
  • login() - verifica as credenciais do usuario no banco de dados
  • atualizar() - atualiza os dados do usuario no banco de dados

3.2 Classe Eventos

Arquivo: model/eventos.php

Representa um evento do clube.

Atributos:

  • $nome - nome do evento
  • $data - data em que o evento ocorrera
  • $local - local onde o evento sera realizado

Metodos:

  • __construct($wnome, $wdata, $wlocal) - construtor da classe, inicializa os atributos
  • criarEvento() - insere um novo evento no banco de dados
  • listarEventos() - retorna todos os eventos cadastrados
  • atualizarEventos() - atualiza os dados de um evento no banco de dados

3.3 Classe Cargos

Arquivo: model/cargos.php

Extende a classe Usuario. Representa um usuario com cargo especifico, adicionando permissoes de administracao ao sistema.

Atributos proprios (alem dos herdados de Usuario):

  • $id_cargo - identificador do cargo do usuario
  • $data - data de atribuicao do cargo

Metodos:

  • __construct($wnome, $wsexo, $wid, $wsenha, $wid_cargo, $wdata) - construtor que chama o parent::__construct() de Usuario e inicializa os atributos proprios
  • excluirUsuario() - remove um usuario do banco de dados (apenas Admin)
  • excluirEvento() - remove um evento do banco de dados (apenas Admin)

Relacao entre as classes:

    Usuario
       |
    (extends)
       |
    Cargos

4. Tabelas do banco de dados

Nome do banco: ClubeSocial

4.1 Tabela: usuario

Campo       Tipo            Descricao
---------   -----------     ----------------------------
id          VARCHAR(30)     Chave primaria - definido pelo usuario
nome        VARCHAR(100)    Nome completo do membro
sexo        CHAR(1)         M ou F
senha       VARCHAR(20)    Senha
id_cargo    INT             Chave estrangeira - referencia tabela cargo

4.2 Tabela: cargo

Campo       Tipo            Descricao
---------   -----------     ----------------------------
id_cargo    INT             Chave primaria - gerada automaticamente
nome        VARCHAR(50)     Nome do cargo (ex: Admin, Membro)
data        DATE            Data de atribuicao do cargo

4.3 Tabela: evento

Campo       Tipo            Descricao
---------   -----------     ----------------------------
id_evento   INT             Chave primaria - gerada automaticamente
nome        VARCHAR(100)    Nome do evento
data        DATE            Data do evento
local       VARCHAR(100)    Local onde ocorre o evento
